---
name: generate-daily-report
description: 根据工作记录生成日报。适用于日常开发进展、任务完成情况、阻塞项和下步计划，不适用于周报、月报或会议纪要。
---

# 目标
把零散的工作记录整理成适合团队阅读的日报。

# 何时使用
当用户要求“生成日报”“整理今天工作”“输出工作日志”“写日终汇报”时使用。

# 不要使用的情况
- 用户明确要周报或月报
- 用户要会议纪要
- 用户要技术方案
- 用户只是做简单问答

# 输入
通常包含：
- 日期
- 工作内容
- 阻塞项
- 计划
- 备注（可选）

# 输出要求
建议按以下结构输出：
1. 今日完成
2. 进行中事项
3. 阻塞与风险
4. 明日计划

# 处理步骤
1. 识别工作内容中的已完成事项。
2. 将正在推进的事项单独列出。
3. 提炼阻塞项和风险。
4. 生成简洁、专业、适合直接发送给团队的日报。

# 约束
- 语气简洁、专业。
- 不要写空话和模板套话。
- 不要扩写用户没有提供的内容。
- 重点突出进展、结果和阻塞。
